# История проекта (Project History)

Проект прошел путь от исследования и командного прототипа до независимой разработки универсальной, воспроизводимой системы мультимодального поиска.

### Этап 1: Предварительное исследование
* **Цель**: Изучение и теоретический разбор подходов, описанных в статье *«Video-RAG: Visually-aligned Retrieval-Augmented Long Video Comprehension»*.
* **Результаты**:
  - Обнаружение и анализ концепта Video-RAG, выделение ключевых стадий (разбор запроса, мультимодальный поиск по временным окнам, формирование ответа с таймкодами).
  - Сборка и подготовка демонстрационных видеофрагментов (выполнено одногруппницей РАНХиГС).

### Этап 2: Прототип для AI CHAMP 2026
* **Цель**: Разработка первичной системы поиска по видео в рамках участия в хакатоне AI CHAMP (репозиторий [AI_champ_OKKO](https://github.com/maxlyara1/AI_champ_OKKO)).
* **Результаты**:
  - Создана первая прототипная система поиска по перекрывающимся окнам (30/10, 59/15, 90/30).
  - Прототип уже содержал обработку речи, текста на экране и кадров, векторный поиск и веб-интерфейс.
  - Решение было жестко привязано к закрытым конкурсным данным, интерфейсам и метаданным платформы Okko.

### Этап 3: Учебная переработка — video-rag-research
* **Цель**: Устранение привязки к конкурсу и адаптация под любые видеофайлы в качестве учебного проекта (репозиторий [video-rag-research](https://github.com/maxlyara1/video-rag-research)).
* **Результаты**:
  - Код полностью очищен от закрытых внешних интерфейсов и конкурсных метаданных.
  - Система адаптирована под автономную работу с любыми русскоязычными видеофайлами.
  - Реализовано кэширование извлеченных признаков в локальные файлы JSON для возможности быстрого перестроения индексов.

### Этап 4: Подготовка к Junior ML Contest — Репозиторий multimodal-video-search
* **Цель**: Независимое построение финальной воспроизводимой системы к Junior ML Contest 2026.
* **Результаты**:
  - Разработан универсальный воспроизводимый конвейер поиска (`VideoRAGPipeline`) в текущем репозитории.
  - Заменен и локализован технологический стек: интегрирована локальная модель векторных представлений `Qwen3-Embedding-0.6B` с ускорением на Apple Silicon MPS, EasyOCR для текста на кадрах, BLIP-base с извлечением сущностей и отношений через spaCy.
  - Разработаны единый управляющий модуль координации, асинхронная серверная часть на FastAPI с SSE-трансляцией логов и новый веб-интерфейс плеера на чистом HTML/JS/CSS.
  - Реализован проверочный набор (выборка для настройки) на 50 запросах. Проведено сравнение способов объединения результатов (Sum, RRF, Max-per-Modality) с выбором активных модальностей.
  - Подготовлен экспериментальный Dockerfile для запуска.
