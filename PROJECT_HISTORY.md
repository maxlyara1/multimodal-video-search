# История проекта (Project History)

Проект прошел путь от исследования и командного хакатонного прототипа до независимой разработки универсальной, воспроизводимой системы мультимодального поиска.

### Этап 1: Предварительное исследование (До марта 2026)
* **Цель**: Изучение и теоретический разбор подходов, описанных в статье *«Video-RAG: Visually-aligned Retrieval-Augmented Long Video Comprehension»*.
* **Результаты**:
  - Обнаружение и анализ концепта Video-RAG, выделение ключевых стадий (декомпозиция запроса, мультимодальный поиск по временным окнам, генерация ответа с таймкодами).
  - Сборка и подготовка демонстрационных видеофрагментов (выполнено одногруппницей РАНХиГС).

### Этап 2: Хакатонный прототип — AI CHAMP 2026 (Март 2026)
* **Цель**: Разработка первичной системы поиска по видео на хакатоне AI CHAMP (репозиторий [AI_champ_OKKO](https://github.com/maxlyara1/AI_champ_OKKO)).
* **Результаты**:
  - Создана первая прототипная система поиска по перекрывающимся окнам (30/10, 59/15, 90/30).
  - Включала Whisper ASR, EasyOCR, генерацию текстовых описаний кадров на Qwen-VL, векторное хранилище Qdrant с эмбеддингами BGE-M3, слияние RRF и локальный реранкер.
  - Реализован FastAPI-бэкенд и интерфейс на React/Vite.
  - Решение было жестко привязано к закрытым конкурсным данным, API и метаданным платформы Okko.

### Этап 3: Учебная переработка — video-rag-research (Апрель 2026)
* **Цель**: Очистка решения от доменной специфики Okko и адаптация под любые видеофайлы в качестве учебного проекта (репозиторий [video-rag-research](https://github.com/maxlyara1/video-rag-research)).
* **Результаты**:
  - Код полностью очищен от закрытых внешних API и конкурсных метаданных.
  - Пайплайн адаптирован под автономную работу с любыми русскоязычными видеофайлами.
  - Реализовано кэширование извлеченных признаков в локальные JSON-артефакты для возможности быстрого перестроения индекса.

### Этап 4: Конкурсная версия — multimodal-video-search (Июль 2026)
* **Цель**: Независимое построение финального воспроизводимого Video-RAG пайплайна к Junior ML Contest 2026.
* **Результаты**:
  - Разработан универсальный воспроизводимый конвейер поиска (`VideoRAGPipeline`) в репозитории [multimodal-video-search](https://github.com/maxlyara1/multimodal-video-search).
  - Заменен и локализован технологический стек: интегрирована локальная эмбеддинг-модель `Qwen3-Embedding-0.6B` с ускорением на Apple Silicon MPS, EasyOCR для текста на кадрах, BLIP-base с извлечением сущностей и отношений через spaCy.
  - Разработаны единый оркестратор, асинхронный FastAPI-бэкенд с SSE-трансляцией логов и новый веб-интерфейс плеера на чистом HTML/JS/CSS.
  - Реализован валидационный бенчмарк (development/validation set) на 50 запросах. Проведено сравнение стратегий слияния модальностей (Sum, RRF, Max-per-Modality) с Query Routing.
  - Подготовлена Docker-контейнеризация для воспроизводимого запуска.
