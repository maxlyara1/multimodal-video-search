# Результаты тестирования поиска по видео (Benchmark)

Сравнение стратегий поиска и алгоритмов слияния результатов:
- **ASR-only**: поиск только по аудиодорожке (Baseline)
- **Multimodal (Sum)**: гибридный поиск с простым суммированием сырых косинусных расстояний
- **Multimodal (RRF)**: гибридный поиск с ранговым слиянием Reciprocal Rank Fusion
- **Multimodal (Max-per-Modality)**: гибридный поиск с объединением взвешенного максимума по каждой модальности

## 4. Анализ и выводы

1. **Преимущество Max-per-Modality и Query Routing**: На текущем development-наборе с замороженной декомпозицией запросов стратегия взвешенного максимума Max-per-Modality в сочетании с Query Routing показала наилучший результат: Hit@1 вырос до 80.00% (прирост на 20% по сравнению с baseline 60.00%), а Hit@3 достиг 100.00% (прирост на 40% по сравнению с baseline 60.00%). Данный результат является предварительным и требует подтверждения на независимой репрезентативной выборке.
2. **Влияние RRF и Sum**: Переход к мультимодальному поиску с Query Routing даже при простом суммировании косинусных расстояний (Sum) превосходит ASR-only baseline, обеспечивая Hit@1 = 70.00%. Использование рангового слияния RRF компенсирует разницу шкал и повышает Hit@1 до 80.00%, но уступает Max-per-Modality по Hit@3 (90.00% против 100.00%) из-за вытеснения релевантных кандидатов в нижнюю часть выдачи при фиксированном слиянии.
3. **Задержка поиска (Local Latency)**: Средняя задержка локального поиска (включая векторизацию и слияние) составляет для baseline 0.388s, а для мультимодальных режимов колеблется в диапазоне от 0.554s до 0.579s.
